# 协作工作流（强制执行）

> 目标：避免因为读到旧信息而回写旧版本；避免忘记提交导致回滚丢功能。

## 单一真源原则

- 现状以 `src/` 与 `src-tauri/` 代码为准。
- 文档唯一真源在 `docs/`（从 `docs/INDEX.md` 开始）。

## 改动节奏（必须）

- 每次改动前：先全局搜索定位入口，禁止凭空重写。
- 每次改动后：必须同步更新 `docs/PROJECT_STATUS.md`：
  - 改了什么
  - 影响哪些文件
  - 如何验证（最少写 1 条）

## Git 提交（必须）

- 每次可运行的改动都必须提交并推送：
  - `git add -A`
  - `git commit -m "..."`
  - `git push`

## 禁止事项

- 禁止把真实 key/密码/个人隐私写进仓库。
- 禁止用过期文档作为功能现状依据（必须以代码为准）。
